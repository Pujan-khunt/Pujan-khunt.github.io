#  Everything in JavaScript happens in Execution Context

Execution Context comprises of two compartments
1. Memory Allocation
2. Code Execution  

<br/>

## Memory Allocation
- The memory part stores all the values of the variables and other memory related stuff, like key value pairs, value of variables, functions etc.

- The fancy and heavy word for this memory compartment is **Variable Environment**.

<br/>

## Code Execution
- The code execution part handles the line by line execution of the JavaScript code.

- The fancy and heavy word for this Code Execution compartment is **Thread of Execution.**#  Everything in JavaScript happens in Execution Context

# JavaScript is a synchronous single-threaded language

**Synchronous**: Meaning JS can only execute 1 command/line at a time  
**Single Threaded**: Meaning JS cant go the next line until it has finished executing the current line.
